# Trace ID, Span ID, and Request ID — Agent Mesh

## Overview

Every agent invocation in the mesh writes three correlation IDs to the audit trail and logs. Understanding what each one covers and when it is created is essential for debugging multi-agent request flows.

---

## `trace_id`

| Property | Detail |
|---|---|
| **Created by** | OpenTelemetry (OTel) SDK — automatically, not by application code |
| **When** | Once per top-level user request, when the root span is opened in the orchestrator |
| **Format** | 32-character hex string (128-bit) |
| **Scope** | The entire request tree — every agent called within one user query shares the same `trace_id` |
| **Where read** | `src/middleware/audit_middleware.py` — `_trace_ids()` reads it from the active OTel span context |

```python
# src/middleware/audit_middleware.py
def _trace_ids() -> tuple[str, str]:
    """Returns (trace_id, span_id) of the active span, or ('-', '-').
    Lets each audit record correlate with the distributed trace emitted by the
    SDK's AgentTelemetryLayer for the same agent run.
    """
    ctx = trace.get_current_span().get_span_context()
    return format(ctx.trace_id, "032x"), format(ctx.span_id, "016x")
```

**Key point:** A new `trace_id` is created for each user query. All agents (ComplianceAgent, DataAgent, RAGAgent, PriceAssistAgent) that serve that query share the same `trace_id`.

---

## `span_id`

| Property | Detail |
|---|---|
| **Created by** | OpenTelemetry SDK — automatically, one per operation |
| **When** | Each time an agent starts processing; tool calls and LLM calls also get child spans |
| **Format** | 16-character hex string (64-bit) |
| **Scope** | A single agent invocation — unique per row in the audit trail |
| **Where read** | Same `_trace_ids()` function as `trace_id` |

**Key difference from `trace_id`:** `trace_id` = the whole conversation turn; `span_id` = one agent's work within that turn. Every row in `audit_trail.jsonl` has a different `span_id`.

---

## `request_id`

| Property | Detail |
|---|---|
| **Created by** | Application code in `src/mesh/orchestrator.py` |
| **When** | Once per user query, at the start of `handle_request()` |
| **Format** | 8-character uppercase hex string (e.g., `"A1B2C3D4"`) |
| **Scope** | Per-request — same as `trace_id` but application-level, independent of OTel |
| **Propagation** | W3C Baggage header `fab.request_id` — travels to all remote A2A agents automatically |

```python
# src/mesh/orchestrator.py
request_id = uuid.uuid4().hex[:8].upper()
```

**Why it exists alongside `trace_id`:** `request_id` is lighter weight, human-readable, and survives in contexts where OTel context may not be available. It is also suitable for displaying in UIs or error messages.

---

## How They Relate — Worked Example

For the query `"Pricing recommendation for CUST002 and When must a bank notify CBUAE about an AI model incident?"`, all agents share one `trace_id` while each gets its own `span_id`:

```
trace_id: 6d0d9a00d9bc7de1a2bfb9438be7cd02   ← same for all agents below
  │
  ├─ span_id: 0b0220280335b47e  →  ComplianceAgent   (safety check)
  ├─ span_id: 394b466c05aab271  →  DataAgent          (pricing data)
  ├─ span_id: ce4a1de266cdeddc  →  RAGAgent           (CBUAE policy lookup)
  ├─ span_id: af457bd317712cf5  →  DataAgent          (second call)
  ├─ span_id: abe16ab3ceb4b2a9  →  DataAgent          (third call)
  └─ span_id: 07e6bc3fcbd32efc  →  PriceAssistAgent   (final synthesis)
```

Each new user query → fresh `trace_id` + fresh `request_id`. Each agent invocation within that query → fresh `span_id`.

## Summary Table

| ID | Generated by | Format | Scope | Propagation mechanism |
|---|---|---|---|---|
| `trace_id` | OTel SDK (automatic) | 32-char hex | Per-request (whole query tree) | W3C `traceparent` header |
| `span_id` | OTel SDK (automatic) | 16-char hex | Per-agent-operation | Implicit in OTel context |
| `request_id` | `uuid4().hex[:8].upper()` in orchestrator | 8-char uppercase hex | Per-request (application level) | W3C Baggage `fab.request_id` |

## Key Source Files

| File | Role |
|---|---|
| `src/mesh/orchestrator.py` | Creates `request_id`; opens root span (which creates `trace_id`) |
| `src/middleware/audit_middleware.py` | Reads `trace_id` + `span_id` from OTel context; writes to audit trail |
| `src/observability/baggage.py` | Stores/retrieves `request_id` (and user/role/session) via W3C Baggage |
| `src/observability/logging_config.py` | Injects all three IDs into every log record via `TraceContextFilter` |
| `src/a2a/hosting.py` | Extracts `traceparent` + baggage from inbound A2A requests |
| `src/observability/setup.py` | Configures OTel SDK and W3C propagators |
