## Running the Pipeline

All scripts must be run **in order** from the project root directory.

### Step 1 – Validate raw data

```bash
python 01_validate_raw_data.py
```

Checks each file in `data/raw/` for:
- Expected column presence
- Missing values
- Duplicate primary-key records
- Numeric and date type validity

Review the console log for any `ERROR` or `WARNING` lines before proceeding.

---

### Step 2 – Create curated data (CSV)

```bash
python 02_create_curated_data.py
```

Reads every CSV from `data/raw/`, applies cleaning rules, and writes cleaned files to `data/curated/`.

Cleaning applied per file:
| Check | Action |
|---|---|
| Duplicate rows | Drop, keep first |
| Numeric columns | Cast to `float`, coerce errors to `NaN` |
| Date columns | Parse with `pd.to_datetime` |
| String columns | Strip whitespace |
| Remaining NaN (numeric) | Fill with column **median** |
| Remaining NaN (string) | Fill with `"Unknown"` |

---

### Step 3 – Create semantic layer (CSV, optional)

```bash
python 03_create_semantic_layer.py
```

Reads curated CSVs and builds five business views in `data/semantic/`:

| File | Description |
|---|---|
| `customer_360.csv` | Customer master enriched with aggregated deal KPIs (volume, win rate, avg margin) |
| `pricing_recommendation_view.csv` | Each deal annotated with product details, matching policy, and compliance flags |
| `margin_analysis.csv` | Deal-level margin decomposition (net margin, spread over benchmark, vs. recommended) |
| `profitability_summary.csv` | Profitability roll-up by customer × product type with tier classification |
| `rwa_impact_view.csv` | RWA-weighted exposure, capital required, and return-on-RWA per won deal |

---

### Step 4 – Load curated data into MySQL

> **Prerequisite:** MySQL must be running and `.env` must contain the correct password.

```bash
python 03_load_curated_to_mysql.py
```

What it does:
1. Creates schemas `fab_curated` and `fab_semantic` (runs `sql/01_create_schemas.sql`).
2. Creates the five curated tables (runs `sql/02_create_curated_tables.sql`).
3. Loads each curated CSV into its corresponding `fab_curated` table using pandas + SQLAlchemy.

---

### Step 5 – Create semantic views in MySQL

```bash
python 04_create_semantic_views.py
```

Executes `sql/03_create_semantic_views.sql` to create five `CREATE OR REPLACE VIEW` statements in `fab_semantic`. Runs a row-count smoke-test on each view and prints the results.

After this step the MCP server (or any BI tool) should connect to MySQL and query **only** the `fab_semantic` schema.

## Key Columns Reference

### customer_360.csv
| Column | Description |
|---|---|
| `customer_id` | Unique customer identifier |
| `customer_segment` | SME / Corporate |
| `total_deals` | Total number of deals |
| `won_deals` | Number of won deals |
| `win_rate_pct` | Win rate (%) |
| `total_deal_volume_aed` | Sum of deal requested amounts |
| `avg_expected_margin_pct` | Average expected margin |

### pricing_recommendation_view.csv
| Column | Description |
|---|---|
| `policy_compliant` | `True` if deal meets all policy thresholds |
| `price_below_policy_floor` | `True` if approved price < policy min margin |
| `margin_below_min` | `True` if expected margin < policy minimum |
| `discount_exceeds_policy` | `True` if relationship discount exceeds policy cap |

### margin_analysis.csv
| Column | Description |
|---|---|
| `net_margin_pct` | Approved price − funding cost − risk premium − discount |
| `spread_over_benchmark_pct` | Approved price − latest treasury benchmark rate |
| `margin_vs_recommended_pct` | Approved price − system recommended price |

### rwa_impact_view.csv
| Column | Description |
|---|---|
| `rwa_aed` | Risk-Weighted Assets in AED |
| `capital_required_aed` | RWA × 8% Basel III capital requirement |
| `return_on_rwa_pct` | Net revenue ÷ RWA × 100 |
| `is_high_rwa` | `True` if RWA weight ≥ 100% |

## MCP Server

The MCP server exposes the five `fab_semantic` views as callable tools for any MCP-compatible client (e.g. Claude Desktop, VS Code Copilot).

### Folder layout

```
mcp_server/
├── __init__.py    # Package marker
├── db.py          # SQLAlchemy engine factory (reads .env, URL-encodes password)
├── tools.py       # Pure query functions — one per semantic view
└── server.py      # FastMCP server — registers tools and starts the server
scripts/
└── test_mcp_db_connection.py   # DB smoke-test (connection + view list + sample query)
```

### MCP Tools

| Tool | View queried | Input |
|---|---|---|
| `customer_360` | `fab_semantic.customer_360` | `customer_id` |
| `pricing_recommendation` | `fab_semantic.pricing_recommendation_view` | `customer_id` |
| `profitability_summary` | `fab_semantic.profitability_summary` | `customer_id` |
| `margin_analysis` | `fab_semantic.margin_analysis` | `customer_id` |
| `rwa_impact` | `fab_semantic.rwa_impact_view` | `customer_id` |

Pass an empty string for `customer_id` to return all records (capped at 100 rows).

### Running the MCP server

```bash
# 1. Verify DB connection first
python scripts/test_mcp_db_connection.py

# 2. Start the MCP server (stdio transport — default for MCP clients)
python -m mcp_server.server
```

### Connecting via VS Code / Claude Desktop

Add the following to your MCP client config:

## AI Agent + Streamlit Chat UI

A LangChain + Groq agent (`agent.py`) answers business questions in natural
language by calling the same semantic-layer query functions used by the MCP
server. A Streamlit GUI (`app.py`) provides a simple chat interface.

| File | Purpose |
|---|---|
| `agent.py` | LangChain Groq agent — wraps `mcp_server/tools.py` functions as 5 tools, exposes `build_agent()` / `run_agent()` |
| `app.py` | Streamlit chat UI — title, chat input, conversation history, calls the agent |

### Agent tools

| Tool | Semantic view |
|---|---|
| `customer_360(customer_id)` | `customer_360` |
| `pricing_recommendation(customer_id)` | `pricing_recommendation_view` |
| `margin_analysis(customer_id)` | `margin_analysis` |
| `profitability_summary(customer_id)` | `profitability_summary` |
| `rwa_impact(customer_id)` | `rwa_impact_view` |

### Required environment variables (`.env`)

```
GROQ_API_KEY=<your_groq_api_key>
MYSQL_USER=root
MYSQL_PASSWORD=<your_mysql_password>
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306

```

## Notes

- All scripts use **standard logging** — run with `python -u script.py` to see unbuffered output in some environments.
- Scripts are designed to be run from the **project root** (`Stru_data_mcp_poc/`). Relative paths (`data/raw`, `data/curated`, `data/semantic`) are resolved from the current working directory.
- The MCP server **and** the AI agent connect **only** to `fab_semantic` — they never touch `fab_curated` or raw tables directly.
- Passwords containing special characters (e.g. `@`) are automatically URL-encoded in `mcp_server/db.py` using `urllib.parse.quote_plus`.
- This is a **POC** — no orchestration framework is required.
